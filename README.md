# Nerfs-pytorch

A pipeline to train [NeRF (Neural Radiance Fields)](http://www.matthewtancik.com/nerf) on **your own video**, running on RTX 5090 (CUDA 12.8 / sm_120).

This repo extends our [PyTorch NeRF port](https://github.com/SonalikaBisoyi/Nerfs-pytorch) with a preprocessing pipeline that converts any video into the LLFF format NeRF expects — using `video2imgs.py` and `imgs2poses.py` from the [LLFF repo](https://github.com/fyusion/llff).

## Pipeline Overview

```
Your video (.mp4 / .mov)
        ↓  video2imgs.py       — extract frames
        ↓  imgs2poses.py       — run COLMAP to estimate camera poses
        ↓  run_nerf.py         — train NeRF on your scene
        ↓  --render_only       — render novel views / fly-through video
```

## Requirements

### 1. Clone this repo
```bash
git clone https://github.com/SonalikaBisoyi/Nerfs-pytorch.git
cd Nerfs-pytorch
```

### 2. Set up the Python environment
```bash
conda env create -f environment.yml
conda activate nerf_pytorch
```

### 3. Install COLMAP

`imgs2poses.py` uses COLMAP under the hood to compute camera poses from your frames.

```bash
# Ubuntu / Debian
sudo apt-get install colmap

# Verify
colmap -h
```

On other platforms, see the [COLMAP installation guide](https://colmap.github.io/install.html).

### 4. Clone the LLFF repo (for preprocessing scripts)
```bash
git clone https://github.com/fyusion/llff.git
```

## Step-by-Step: From Video to NeRF

### Step 1 — Extract frames from your video

```bash
python llff/video2imgs.py <path_to_your_video.mp4> <output_image_folder>
```

Example:
```bash
python llff/video2imgs.py myvideo.mp4 data/my_scene/images
```

**Tips for a good video:**
- Walk slowly around the object or scene, keeping it in frame
- 30–60 seconds at 30fps is usually enough (900–1800 frames)
- Avoid fast motion, motion blur, or changing lighting
- Keep overlap between frames high (~80%)

---

### Step 2 — Compute camera poses with COLMAP

```bash
python llff/imgs2poses.py <output_image_folder>
```

Example:
```bash
python llff/imgs2poses.py data/my_scene
```

This runs COLMAP's Structure-from-Motion pipeline and produces a `poses_bounds.npy` file inside your scene folder. This file contains the camera poses and scene bounds that NeRF needs.

Your scene folder should now look like:
```
data/my_scene/
├── images/          ← extracted frames
│   ├── frame_0001.png
│   ├── frame_0002.png
│   └── ...
└── poses_bounds.npy ← generated by imgs2poses.py
```

---

### Step 3 — Create a config file

Create a file `configs/my_scene.txt`:

```text
expname = my_scene_test
basedir = ./logs
datadir = ./data/my_scene

dataset_type = llff
factor = 8
llffhold = 8

N_rand = 1024
N_samples = 64
N_importance = 128

use_viewdirs = True
raw_noise_std = 1e0

lrate = 5e-4
lrate_decay = 250
N_iters = 200000
```

For your RTX 5090, you can push the batch size harder:
```text
N_rand = 4096
chunk = 131072
netchunk = 262144
```

---

### Step 4 — Train NeRF

```bash
python run_nerf.py --config configs/my_scene.txt
```

After training for 100k iterations (~1–2 hours on RTX 5090), you can find the following video at `logs/my_scene_test/my_scene_test_spiral_100000_rgb.mp4`:

![](https://user-images.githubusercontent.com/7057863/78473103-9353b300-7770-11ea-98ed-6ba2d877b62c.gif)

### Step 5 — Render novel views

Once training is complete (or from a checkpoint):

```bash
python run_nerf.py --config configs/my_scene.txt --render_only
```

## Expected Training Time (RTX 5090)

| Resolution (`factor`) | Iterations | Estimated Time |
|-----------------------|-----------|---------------|
| factor=8 (low-res, fast) | 200k | ~2–3 hours |
| factor=4 (medium) | 200k | ~6–8 hours |
| factor=1 (full res) | 200k | ~12–16 hours |

Start with `factor=8` to validate your scene, then retrain at higher resolution.

## Troubleshooting

| Problem | Fix |
|---------|-----|
| `poses_bounds.npy not found` | `imgs2poses.py` did not finish — check COLMAP is installed and rerun |
| COLMAP fails / sparse reconstruction empty | Your video may have too much motion blur or not enough overlap between frames |
| `No images found in images_8` | Your `factor` setting doesn't match the folder that exists — change `factor` or run the downsampling step manually |
| `CUDA out of memory` | Reduce `--chunk` and `--netchunk` in your config |
| Blurry / noisy output | Try longer training, more overlap in your video, or reduce `factor` |
